# 用户使用流程 · 用户端（小程序）

> 版本：V1.0　日期：2026-06-05
> 范围：新菘福减重小程序用户端，按场景拆解的使用流程
> 图示：Mermaid 流程图（GitHub / Typora / VSCode 可直接渲染）
> 关键状态：**减重状态**（暂不减重 / 方案执行中 / 已达标 / 已暂停 / 已流失）× **服务状态**（未加入·无营养师 / 已加入·有专属营养师）

---

## 0. 全局状态机（用户生命周期）

```mermaid
flowchart LR
  A[微信注册/登录] --> B[完成基础信息]
  B --> C{是否需要减重?}
  C -- 暂不减重 --> D[暂不减重态<br/>仅记录体重·无方案]
  C -- 需要减重 --> E[生成减重方案]
  E --> F[确认按方案执行]
  F --> G[方案执行中]
  D -- 设置减重目标激活 --> E
  G -- 达成目标 --> H[已达标]
  G -- 长期不活跃 --> I[已暂停/已流失]
  G -. 加入减重大家庭 .-> J[已加入·有专属营养师]
  D -. 加入减重大家庭 .-> J
```

- **服务状态**与减重状态正交：无论是否减重，用户初始都是「未加入·无营养师」；加入减重大家庭后变「已加入」。
- 后台据此分配专属营养师、推送营养师报告、开通评餐与在线咨询。

## 1. 场景一：首次进入 · 登录引导（8 步）

```mermaid
flowchart TD
  S0[微信授权登录 + 手机号] --> S1[选择性别]
  S1 --> S2[出生日期·算年龄]
  S2 --> S3[身高 / 体重 → 实时BMI分级]
  S3 --> S5[BMI 评估结果<br/>偏瘦/正常/超重/肥胖 + 健康体重区间]
  S5 --> Q{底部选择}
  Q -- 暂不需要减重 --> H1[直接进入首页·暂不减重态]
  Q -- 需要减重 --> S6[设定目标体重<br/>周期由系统按年龄/性别/BMI/差距自动生成]
  S6 --> S7[完善健康信息<br/>4开关:减重药多条+长期用药 / 疾病 / 特殊状态 / 过敏]
  S7 --> S8[生成量化减脂方案<br/>饮食·运动·睡眠·饮水]
  S8 --> S9[确定，按方案打卡]
  S9 --> H2[进入首页·方案执行中]
```

**关键规则**
- 目标 BMI<18 弹偏瘦风险提示；用药区脚注"平台不提供药物推荐，仅作健康档案记录"。
- 选"暂不减重"跳过目标/方案步骤，进入无方案首页（见场景二）。

## 2. 场景二：暂不减重用户 · 后续激活减重方案

```mermaid
flowchart TD
  A[首页·暂不减重态<br/>体重卡:当前 + 健康上下限] --> B[点「设置减重目标」]
  B --> C[设定目标体重弹窗]
  C --> D[完善健康信息弹窗<br/>疾病/用药/特殊状态/过敏]
  D --> E[方案生成中…]
  E --> F[专属减重方案页<br/>概览+饮食+运动+睡眠+饮水]
  F --> G{用户确认}
  G -- 确认，按此方案执行 --> H[转「方案执行中」<br/>首页切换为有方案态]
  G -- 稍后再说 --> A
```

## 3. 场景三：每日打卡（核心高频）

```mermaid
flowchart TD
  H[首页·每日打卡卡] --> CAL{选择日期}
  CAL -- 今天 --> T[当日打卡]
  CAL -- 左右滑动选过去日 --> M[补卡态:提示"正在为X月X日补卡"]
  T --> P{选择打卡项}
  M --> P
  P --> P1[🍽️ 饮食热量 → 见场景四]
  P --> P2[⚖️ 体重:手动/蓝牙秤/拍照 + 体脂 + 三围]
  P --> P3[📏 三围:腰必填/臀/腹]
  P --> P4[🧪 尿酮 → 见场景五]
  P --> P5[🚽 排便 → 见场景六]
  P --> P6[😴 睡眠 / 🏃 运动 / 💧 饮水 / 🩸 生理期]
  P1 & P2 & P3 & P4 & P5 & P6 --> R[保存 → 刷新打卡徽章/状态色点]
```

**关键规则**
- 日期条最近 14 天可滑动，状态色点：合适(绿)/吃多(红)/吃少(琥珀)/未打卡(无)。
- 补卡受后台「补卡规则」约束（允许补卡天数/是否计入达标）。

## 4. 场景四：饮食记录 · 选食物 / AI 拍照

```mermaid
flowchart TD
  D[饮食记录页] --> E{录入方式}
  E -- 餐次入口 --> M{选择餐次}
  M -- 早/午/晚餐 --> F[选择食物页]
  M -- 加餐 --> M2[先选 早加餐/午加餐/晚加餐]
  M2 --> F
  F --> F1[分类:常用/家常菜/肉类/蔬菜/水果…<br/>食材按100g · 家常菜成品菜按份]
  F1 --> F2[点食物 → 克数/份量 → 添加]
  F2 --> F3[可切餐次跨餐批量添加 → 完成]
  E -- 📷 拍照 --> C[AI 拍照识别页]
  C --> C1[识别结果:热量+碳水/蛋白/脂肪+逐项食物<br/>可人工修正]
  C1 --> C2[选择日期 + 餐次]
  C2 --> C3[一键记录到所选餐次]
  F3 & C3 --> S[FS_DATA 单一数据源<br/>首页角标/详情分组/汇总联动刷新]
```

## 5. 场景五：尿酮打卡（AI 识别）

```mermaid
flowchart TD
  A[打开尿酮录入] --> B[拍摄/上传试纸照片]
  B --> C[AI 识别<br/>兼容各品牌试纸·不依赖固定色阶]
  C --> D[输出:未入酮/已入酮/深度入酮 + 置信度]
  D --> E{识别是否准确?}
  E -- 准确 --> G[保存]
  E -- 有误 --> F[手动校正为 未/已/深度入酮<br/>描述随之更新]
  F --> G
  G --> H[深度入酮 → 提示补水/咨询]
```

> 因各厂家试纸色阶不统一，改由 AI 拍照判定；低置信度后台会提示人工校正。

## 6. 场景六：排便打卡（按次记录）

```mermaid
flowchart TD
  A[打开排便录入] --> SW[是否在服用果蔬粉?<br/>开启后持久不自动关 · ⓘ可看说明]
  SW --> B[每排一次记一条]
  B --> C[本次时间:默认当前真实时间·可修改]
  C --> D[选 布里斯托性状 / 颜色 / 异常]
  D --> E[+ 记录本次排便 → 加入当日列表]
  E --> F{今天还有?}
  F -- 是 --> B
  F -- 否 --> G[完成 · 今日次数=记录条数]
  G --> H[黑/红/带血 → 就医提示;单条可删除]
```

## 7. 场景七：营养师页 · 加入减重大家庭

```mermaid
flowchart TD
  N[营养师 Tab] --> ST{服务状态}
  ST -- 未加入·无营养师 --> U1[加入减重大家庭引导卡<br/>专属营养师1对1 + 专属营养方案]
  U1 --> U2[我的减重方案·系统生成]
  U1 --> J[点「去加入」]
  J --> J1[企业微信二维码<br/>长按识别/扫码添加专属减重顾问]
  J1 --> J2[顾问1对1匹配专属营养师]
  J2 --> ST2[转「已加入·有专属营养师」]
  ST -- 已加入·有专属营养师 --> A1[专属营养师卡:发送消息]
  A1 --> A2{评餐 / 方案报告 切换}
  A2 -- 评餐 --> A3[营养师评餐:点评/回复]
  A2 -- 方案报告 --> A4[营养师减重报告 + 我的减重方案]
```

**关键规则**
- 未加入用户：只有系统生成的「我的减重方案」，**无系统单独报告**；评餐/营养师报告/在线咨询都属加入后服务。
- 加入即企业微信引流 → 后台分配专属营养师。

## 8. 场景八：健康预警 → 联系营养师 / 加入

```mermaid
flowchart TD
  H[首页·健康预警卡<br/>命中规则才显示:减重过快/尿酮高/体重反弹…] --> B[点「获取专属营养师」]
  B --> P{服务状态}
  P -- 未加入 --> U[加入减重大家庭弹窗<br/>风险提示 + 企业微信二维码]
  P -- 已加入 --> A[联系营养师弹窗<br/>专属营养师 + 发送消息]
```

## 9. 场景九：数据查看（趋势/详情）

```mermaid
flowchart TD
  H[首页体重卡] --> WD[体重数据详情]
  WD --> WD1[基本信息 + BMI 色条]
  WD --> WD2[减重目标卡<br/>有方案才显示·暂不减重态无此卡]
  WD --> WD3[趋势图:体重/BMI/体脂 × 周/月/年]
  WD --> WD4[身体成分:可编辑展示项]
  D[数据 Tab] --> D1[本周概览:体重/体脂/运动/摄入]
  D1 --> D2[趋势卡网格8项 → 各指标详情]
```

## 10. 场景十：我的 / 健康档案

```mermaid
flowchart TD
  ME[我的 Tab] --> M1[健康数据达标率·圆形进度<br/>饮食/运动/饮水/睡眠]
  ME --> M2[减重咨询]
  ME --> M3[健康档案·单一入口]
  M3 --> M31[基础数据 / 健康数据]
  M3 --> M32[用药记录:可多条 + 长期用药标注]

  M3 --> M34[饮食偏好]
```
