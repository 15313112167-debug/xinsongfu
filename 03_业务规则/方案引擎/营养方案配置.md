如果你的减重小程序后续要支持：

- AI生成饮食方案
- 营养师制定方案
- 不同疾病用户方案
- 减脂/增肌/维持不同目标

那么**个性化营养方案一定不能写死在代码里，而是要做后台配置化。**

------

# **一、整体架构**

```text
用户画像
    ↓
规则引擎
    ↓
营养方案模板
    ↓
个性化调整
    ↓
最终方案
```

例如：

```text
女性
35岁
BMI=30
糖尿病
目标减脂

↓

低GI减脂方案

↓

每日1500kcal
蛋白30%
碳水40%
脂肪30%

↓

生成7天食谱
```

------

# **二、后台配置模块**

## **模块1：用户分层规则配置**

这是方案匹配入口。

### **配置项**

| **配置项**   | **示例** |
| ------------ | -------- |
| 性别         | 男/女    |
| 年龄范围     | 18-30    |
| BMI范围      | 24-28    |
| 体脂率范围   | >30%     |
| 目标类型     | 减脂     |
| 疾病标签     | 糖尿病   |
| 运动水平     | 久坐     |
| 是否用药     | GLP-1    |
| 营养风险等级 | 高       |

------

### **后台效果**

运营配置：

```text
BMI≥28
AND
糖尿病

↓

匹配方案A
```

------

# **模块2：营养方案模板库**

这是核心。

------

## **方案基础信息**

| **字段** | **示例**       |
| -------- | -------------- |
| 方案ID   | PLAN001        |
| 方案名称 | 糖尿病减脂方案 |
| 适用人群 | 超重+糖尿病    |
| 状态     | 启用           |
| 优先级   | 100            |
| 创建人   | 营养师         |

------

## **热量配置**

### **配置**

| **字段**     | **示例** |
| ------------ | -------- |
| 每日热量     | 1500kcal |
| 热量计算方式 | 固定值   |
| 是否动态调整 | 是       |

------

也可以支持：

```text
基础代谢 × 1.2 - 500
```

------

## **三大营养素配置**

### **配置项**

| **项目** | **示例** |
| -------- | -------- |
| 蛋白质   | 30%      |
| 碳水     | 40%      |
| 脂肪     | 30%      |

------

或者：

```text
蛋白
1.5g/kg

碳水
120g

脂肪
50g
```

# **模块3：疾病限制规则**

这是未来业务最容易调整的地方。

必须后台化。

------

## **糖尿病**

配置：

```text
禁用：
高GI食物

推荐：
全谷物
高纤维食物
```

------

## **痛风**

配置：

```text
禁用：
动物内脏
浓肉汤

限制：
海鲜
啤酒
```

------

## **高血压**

配置：

```text
钠摄入<5g
```

------

### **建议设计**

```text
疾病
↓
限制规则
↓
推荐规则
↓
提示规则
```

运营可以随时修改。

# **三、动态调整规则**

减重产品一定会遇到：

```text
用户体重不下降
```

怎么办？

------

建议后台配置：

## **调整规则**

| **条件**     | **动作** |
| ------------ | -------- |
| 7天无变化    | -100kcal |
| 连续增重     | 调整碳水 |
| 掉肌肉       | 增加蛋白 |
| 体脂下降缓慢 | 调整运动 |

------

例如：

```text
连续14天体重无变化

↓

热量减少100kcal
```

------

# **四、营养师干预配置**

建议预留。

------

## **自动介入规则**

| **条件**   | **动作**   |
| ---------- | ---------- |
| BMI>35     | 营养师跟进 |
| 快速减重   | 人工审核   |
| 糖尿病用户 | 强制介入   |
| 用药用户   | 强制介入   |

------

# **五、方案版本管理（容易遗漏）**

业务后面一定会改方案。

建议设计：

| **字段** |
| -------- |
| 版本号   |
| 生效时间 |
| 失效时间 |
| 创建人   |
| 修改记录 |

------

例如：

```text
低碳减脂方案

V1.0
蛋白25%

V2.0
蛋白30%
```

历史用户不受影响。

------

# **六、针对减重小程序，我最推荐的后台配置模型**

```text
① 用户标签规则
        ↓

② 营养方案模板
        ↓

③ 疾病限制规则
        ↓

④ 食物库
        ↓

⑤ 食谱模板
        ↓

⑥ 动态调整规则
        ↓

⑦ 营养师介入规则
```
