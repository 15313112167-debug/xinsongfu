## 沟通建议顺序

建议按照 **"影响开发阻塞 → 影响产品设计 → 影响后续迭代"** 的顺序推进沟通。

## 一、异常提醒具体阈值（最高优先级）

### 背景
会议纪要中明确"异常提醒规则待确认"，这是规则引擎的核心输入，无此规则编码无法进行。

### 需要业务方确认的问题

| 异常场景 | 需确认内容 | 建议默认值 |
|---------|-----------|-----------|
| 体重反弹 | 反弹多少kg定义为异常？ | ≥2kg |
| 长期不打卡 | 连续几天未打卡触发提醒？ | ≥3天 |
| 极端减重 | 每周减重超过多少kg算异常？ | >5kg/周 |
| 尿酮异常 | 高酮持续几天触发预警？ | ≥2天 |
| 睡眠严重不足 | 睡眠时长低于多少小时算异常？ | <4小时 |
| 体重连续上涨 | 连续上涨几天触发预警？ | ≥3天 |
| 目标不合理 | 目标体重低于BMI多少拦截？ | BMI<18.5 |

### 确认后需明确的输出
- 各异常场景的**具体数值阈值**
- 异常触发后**通知对象**（用户、营养师、管理员）
- 通知**方式**（站内信、微信、短信）
- 是否需要**强提醒**（弹窗 vs 红点）

## 二、营养方案模板具体内容结构


### 需要业务方确认的问题

| 问题 | 选项参考 | 建议 |
|------|---------|------|
| 模板是否包含一日三餐+运动？ | A. 仅饮食 / B. 饮食+运动 / C. 饮食+运动+生活建议 | B（推荐） |
| 饮食方案是否细化到每餐？ | A. 只给建议 / B. 具体到每餐吃什么 | B |
| 是否区分不同方案类型？ | 生酮、轻断食、高蛋白、均衡饮食等 | 是 |
| 模板是否含热量目标？ | 日均热量、碳水/蛋白/脂肪比例 | 是 |
| 是否含禁忌项？ | 某些疾病禁用哪些方案 | 是 |
| 是否支持营养师自定义修改模板？ | 复制模板后编辑再推送 | 是 |

## 三、评价体系详细规则

### 背景
评价规则未定义，影响评价管理和营养师考核模块。

### 需要业务方确认的问题

| 问题 | 选项参考 | 建议 |
|------|---------|------|
| 评价对象是谁？ | A. 营养师 / B. 产品/方案 / C. 两者都评 | C（推荐） |
| 评分方式？ | A. 五星评分 / B. 百分制 / C. 满意/一般/不满意 | A（推荐） |
| 评价维度有哪些？ | 专业度、服务态度、效果、响应速度等 | 建议3-5个维度 |
| 是否允许文字评价？ | 是/否 | 是 |
| 是否需要匿名？ | 是/否 | 否 |
| 评价是否公开显示？ | 是/否 | 否（内部使用） |

| 是否支持机器人自动回复？ | 是/否 |

## 五、健康建议模板内容

### 背景
会议中"健康建议模板由业务方提供，无模板则暂不做"。

### 需要业务方确认的问题

| 问题 | 说明 |
|------|------|
| 业务方是否有现成的建议模板？ | 如果有，请提供完整文案 |
| 如果没有，是否由我们协助设计？ | 需确定由谁出文案 |
| 建议模板按哪些场景分类？ | 运动不足、饮食超标、喝水不足、睡眠不足等 |
| 建议文案是否需要分用户类型？ | 大体重 vs 小体重 vs 偏瘦 |

## 六、其他中优先级确认项

### 6.1 数据统计具体指标
- 统计范围：营养师名下所有客户整体数据？还是营养师自身评餐数据？
- 核心展示指标：打卡率、完成率、异常客户数、减重效果？
- 时间维度：日/周/月/自定义？

### 6.2 营养师考核指标
- 哪些指标适合考核？建议：客户减重效果、评餐响应率、客户满意度、客户留存率
- 是否不同级别营养师不同考核标准？

### 6.3 服务周期时长选项
- 服务周期提供哪些选项？建议：7天/15天/20天/30天/自定义
- 到期后是否自动续费？还是手动续费？
- 到期后用户数据如何处理？只读还是存档？

### 6.4 睡眠数据获取方案
- 小程序无法直接获取系统睡眠数据，是否保留手动录入睡眠功能？
- 或者直接去掉睡眠记录模块？
