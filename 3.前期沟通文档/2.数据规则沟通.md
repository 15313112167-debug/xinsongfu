
# **1. 用户基础信息规则**

## **1.1 基础信息采集规则**

需要确认：

| **项目** | **需要沟通的问题**            |         |
| ------ | ---------------------- | ------- |
| 年龄     | 未成年人是否允许使用？            |         |
| 身高     | 是否允许后期修改？修改后历史BMI是否重算？ | 可以修改，不重 |
| 当前体重   | 单位是否固定kg？是否保留小数？       |         |
| 目标体重   | 最低目标限制是多少？             | BMI     |
| 目标周期   | 是否限制最快减重速度？放二期         | 中国营养协会  |

---

## **1.2 BMI规则**

重点确认：

### **BMI计算公式**

建议明确：

- BMI = 体重(kg) / 身高²(m)
- 是否采用中国标准
- 是否区分成人/青少年（单独公式）

### **BMI分级规则**

建议业务明确：

|**BMI范围**|**状态**|**是否触发建议**|
|---|---|---|
|<18.5|偏瘦|是否提醒|
|18.5~23.9|正常|是否鼓励维持|
|24~27.9|超重|是否进入减重方案|
|≥28|肥胖|是否提示专业干预|

### **BMI相关问题**

重点问：

- BMI是否作为核心指标？
- BMI变化异常是否触发消息提醒（可以，放营养师队列，）
	- bmi大于28，小于18报警，体重保持不变，或变化太大预警
- BMI变化多少算有效？
- BMI是否进入趋势分析？

# **2. 问卷评测规则**

---

## **2.1 目标减重合理性规则**

建议重点确认：

|**问题**|**示例**|
|---|---|
|目标体重最低限制|是否低于健康BMI即禁止|
|周减重上限|每周最多减多少kg|
|周减重过快|是否提醒风险|

建议业务明确：

- 平台推荐的健康减重速度
- 是否允许“激进减重”
- 是否给出风险说明

---

## **2.2 疾病史规则**

重点确认：

| **疾病** | **是否影响方案** |     |
| ------ | ---------- | --- |
| 2型糖尿病  | 是否特殊饮食     |     |
| 高血脂    | 是否限制饮食结构   |     |
| 脂肪肝    | 是否特殊建议     |     |
| 高血压    | 是否限制运动强度   |     |
| 痛风     | 是否限制高蛋白    |     |

需要业务明确：

- 哪些疾病属于高风险用户
- 是否需要营养师强介入
- 是否需要特殊标签
- 是否需要限制某些减重方案

---

## **2.3 用药规则**

重点关注合规：

|**问题**|**需要明确**|
|---|---|
|减重药物|是否记录具体药品|
|是否展示药物建议|有较高风险|
|是否允许业务推荐药品|涉及合规|
|药物用户是否特殊管理|建议有风险标识|

建议明确：

- 是否涉及医疗建议
- 是否需要免责声明
- 是否属于健康管理还是医疗行为

# **3. 体重数据规则**

这是最核心的数据模块。

---

## **3.1 数据来源规则**

建议确认：

|**来源**|**是否可信**|**优先级**|
|---|---|---|
|手动输入|一般|低|
|蓝牙体脂秤|高|高|
|拍照识别|待确认|中|

重点问题：

- 同一天多次记录取哪条？
- 是否保留全部历史？
- 是否允许修改历史记录？
- 异常值如何处理？

---

## **3.2 体重异常规则**

建议重点沟通：

| **场景** | **是否提醒** |
| ------ | -------- |
| 24小时骤降 | 是否异常     |
| 体重连续上涨 | 是否预警     |
| 长时间无变化 | 是否干预     |
| 反弹超过阈值 | 是否通知营养师  |

重点确认：

- 反弹定义
- 平台如何定义“平台期”
- 是否自动推送建议
- 是否需要人工跟进

---

## **3.3 目标达成规则**

|**问题**|**示例**|
|---|---|
|达成比例怎么算|当前减重/目标减重|

# **四、你明天最应该重点问的5个问题**

如果时间有限，建议优先把下面5个问题聊透：

## **1. 什么样的数据算异常？**

因为这决定：

- 提醒逻辑
- 风险机制
- 营养师介入
- 用户体验

---

## **2. 什么规则属于医学规则？**

因为这决定：

- 是否合规
- 是否需要医生
- 是否需要免责声明

---

## **3. 哪些规则需要后台配置化？**

因为后期业务一定会频繁调整。

---

## **4. 哪些建议由系统自动生成，哪些必须人工介入？**

这决定：

- AI能力边界
- 营养师工作量
- 系统复杂度

---

## **5. 平台最终最关注的核心指标是什么？**

比如：

- 体重下降
- 体脂下降
- 用户留存
- 打卡率
- 复购
- 营养师服务效率

因为这会影响整个产品设计方向。
